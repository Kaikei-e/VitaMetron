# デイリーアドバイスへの概日リズム評価の統合

## ADR's STATUS

採用済み (2026-04-08)

## CONTEXT

ADR-014 で導入した概日リズム指標（CHS, コサイナー分析, NPAR, 睡眠タイミング, 夜間 HR 低下）がダッシュボードで数値として確認可能になったが、Ollama LLM が生成するデイリーアドバイスにはこれらのデータが渡されていなかった。

アドバイス生成パイプラインは `_collect_health_context()` でデータを収集し、`build_user_prompt()` で日本語トーキングポイントに変換して LLM に渡す構成。概日リズムがパイプラインに含まれていないため、睡眠タイミング規則性や夜間自律神経パターンに基づく助言が生成できなかった。

## DECISION MAKING

既存パイプラインに概日リズムセクションを追加する。3 箇所を変更:

1. **データ収集**: `circadian_scores` テーブルから CHS・コサイナー・NPAR・睡眠タイミング・夜間 HR dip を取得
2. **プロンプト構築**: `_build_circadian_section()` で 7 トーキングポイント（CHS, 夜間心拍低下, 睡眠中間点, ばらつき, ソーシャルジェットラグ, IS, アクロフェーズ）に変換
3. **システムプロンプト**: 概日リズム指標の解釈ガイド追加

条件付き表示: CHS は confidence > 0.3 の場合のみ、ソーシャルジェットラグは 30 分超の場合のみ表示。

検討した代替案:
- JSON のまま渡す → 4B 量子化モデルでは構造化 JSON の解釈精度が低い
- VRI セクションに統合 → 情報量が多くなり VRI の主要因が埋もれる
- IS/IV/RA 全指標を含める → IV と RA は IS と相関が高く冗長

## RESULTS, EFFECTS

### 変更ファイル

| ファイル | 変更 |
|---|---|
| `ml/app/routers/advice.py` | circadian_scores 取得クエリ追加、コンテキスト辞書に circadian 追加、システムプロンプトに解釈ガイド追加 |
| `ml/app/services/advice_prompt.py` | `_build_circadian_section()` 追加、セクション一覧に「概日リズム」追加 |

### PROS

- LLM が夜間心拍低下・睡眠規則性・ソーシャルジェットラグに基づく具体的な改善提案を生成可能に
- 条件付き表示でデータ不足時の無理な言及を回避
- 既存トーキングポイント形式に統一し、出力品質が安定
- 既存テスト 49 件が全て通過

### CONS, TRADEOFF

- プロンプトのトークン数が増加（約 200-400 文字分）。num_predict (2560) の範囲内
- 4B 量子化モデルの概日リズムに対する医学的理解は限定的。事前解釈済みの日本語で渡すことでリスク軽減

## APPENDIX

LLM 出力での概日リズム言及の確認例:
- 「夜間心拍低下が16.2%と正常範囲内を保っている点からは、自律神経のリズム自体は安定しているようです」
- 「概日リズムを優しく整えてあげてください」
